WebApp Park

A complete highly available cloud for providing Web Application for the higher education sector.

Top Features

Authentication

UWAP performs international cross-federated authentication for users across euducational institusions across Europe.

Single Sign-On with Feide and similar solutions works out of the box, not configuration or coding needed.

Groups and collaboration

Collaboration groups should be setup once, and then re-used across applications. UWAP does this for you, but exposing a users group membership to you without to begin with.

Deployment

With a few clicks you'll be able to deploy your Web Application through a WebDAV .

Your application is hosted in the cloud, on a distributed replicated cloud stoarge, and a bunch of front-ends take care of making your application available for your end users at any time.

Data storage

A highly available replicated NoSQL storage is available at your fingertips. As UWAP authenticate the user for you automatically, you can store per-user JSON data to the storage with a single line of code.

More advanced storage queries are available.

Release Management

Application owners / developers, may setup multiple tagged instances of an application.

With a few clicks a developer may setup a test, pre-prod and production instance, and upgrading pre-prod and prod instances can be done with a few clicks in the application manangement dashboard.

REST Gatekeeper

UWAP performs international cross-federated authentication for users across euducational institusions across Europe.

Single Sign-On with Feide and similar solutions works out of the box, not configuration or coding needed.

If you have a REST service, you give access credentials to the UWAP platform, and you can provision access to the interface through the platform dashboard. REST calls will then be proxied through the platform, and HTTP headers including information about the user and group will be provided to your server. The client-facing side of the platform will offer OAuth 2.0 protected endpoints.

REST Javascript API

For web applications hosted on the platform, a simple javascript API expose access to perform REST calls to remote data providers.

An app owner can through the dashboard administration setup OAuth credentials to connect to other data providers using OAuth 1.0, OAuth 2.0 or custom authorization. The platform will use its built-in storage to store the OAuth access tokens associated with the authenticated users account. This saves the app developer from a signficant hassle, and enables cross-domain authenticated REST communication a pleasure.

This API also offers some convenient features such as XML to JSON translation, etc.

Messaging Service

The platform maintains a list of all users that have accepted to use your application.

You may reach out to all the users of your application through your platform dashboard, or through a REST API.

The user will be asked if he accepts email communcation at the time he is using the application for the first time. The grant usually limits the rate at which an application owner may contact its users. And also important for the user, the app owners never sees the email address, and when the user decides to reject email grant for an applicaiton, the application owner have no way of sending email to this user anymore.

The Web AppStore

All applications running on this platform is listed in the WebApp store, targeting end users. End users may search, and browse through applications.
